Legend: Hospital Core emits events → Analytics/ML derives features, marts, and guarded insights → Patient Edge consumes insights via API and returns engagement/RPM signals. Safety/Policy engine gates anything patient-facing.

Notes that make it “living”

Event heartbeat: Core publishes ADT/Orders/Results/Notes as a stream; synthetic patients keep arriving, being treated, discharged.

Policy shocks: inject scenario switches (e.g., new guideline, staffing change). Analytics/ML re-equilibrates → you observe system impact.

Closed loops: Patient Edge (RPM check-ins, explainers) feeds back engagement/outcomes to the Feature Store; experiments measure lift/harms.

Guardrails first: Safety/Policy engine tags every outward insight with provenance (refresh time, rule/model id) and blocks unsafe messaging.

One “thin thread”: Start with a single care pathway (e.g., ED → inpatient → discharge CHF). Every layer (EHR, analytics, GenAI, patient UI) hangs off that same thread to stay coherent.

What this enables

Swap models without rewiring apps (insights are contracted JSON with provenance).

Test system effects, not just model metrics (A/B harness tracks readmits/ED returns under synthetic shocks).

Stay privacy-clean: all PHI is synthetic; patient-edge artifacts are generated from read-only insights with audit/consent flags.
